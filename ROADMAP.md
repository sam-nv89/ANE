# ROADMAP.md

Дорожная карта проекта **Adaptive Nutrition Engine**.
Мы используем гибкую архитектуру, позволяющую легко добавлять новые функции в ядро алгоритма и расширять пользовательский интерфейс.

## 🚀 Текущий статус: MVP Release 1.0 (Апрель 2026)

### ✅ Завершенные этапы
- [x] Разработка визуального языка (Design System, Dark Theme, CSS variables).
- [x] Проектирование и верстка промо-лендинга (Hero, Engine, Problems, Metrics, TechStack, Footer).
- [x] Проектирование SPA архитектуры, миграция с лендинга.
- [x] Интеллектуальный Онбординг (6 шагов), валидация данных форм.
- [x] Zustand хранилища: `useUserStore` (учет и пересчет КБЖУ, сохранение в localStorage).
- [x] База данных: структура `recipes.json`.
- [x] Алгоритм фильтрации по аллергенам и `timeWindow` пользователя.
- [x] Модуль монотонности `monotonyIndex.js` — per-mealType лимиты блюд.
- [x] Ядро генератора: `generator.js` — умный подбор с 4-уровневым fallbacks и подстройками калорий.
- [x] Dashboard: 7-дневный grid, meal-карточки, SVG кольца прогресса (Белки/Жиры/Углеводы).
- [x] `MealDetail`: Карточка рецепта с шагами, БЖУ и тегами.
- [x] `ShoppingList`: Автосборка ингредиентов (Zero Waste логика, чекбоксы).
- [x] `Export System`: Стабильный экспорт Меню и Списка покупок (PDF/Txt) с поддержкой Stealth-режима (без вспышек).
- [x] `Progress Tracking`: Графики изменения веса, интерактивные подсказки, выбор даты замера и автосинхронизация Выполнения рациона (Compliance).
- [x] `Profile`: Экран переконфигурации биометрии, вызывающий умный пересчет плана.
- [x] `Profile & Settings`: Создан полноценный раздел настроек с управлением аллергенами, предпочтениями и липкой панелью действий. Исправлен контраст и UX.
- [x] `Navigation Stability`: Стабилизированы маршруты и переходы между внутренними разделами приложения.
- [x] `Food Diary 2.0`: Интерактивные отметки съеденного, ручной ввод продуктов, экспорт меню и списка (PDF/Txt).

#### Ядро алгоритмов (Core)
- [x] Динамическое масштабирование граммовок (расчет `totalMultiplier`, скейлинг ингредиентов и порций).
- [x] Алгоритм Smart-Substitution — возможность для пользователя заменить одно предложенное блюдо "на лету" из альтернативного мини-пула.
- [x] Calorie Precision Sync — синхронизация калорий между недельным планом и детальным видом рецепта.
- [ ] Фильтр бюджета — генератор должен уметь оптимизировать Shopping List под выделенный недельный бюджет (собирать блюда с пересекающимися ингредиентами).
